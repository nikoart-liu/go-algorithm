# Go 算法学习路径开发准则 (Product Guidelines)

## 文档编写准则 (Prose Style)
- **通俗易懂**：优先使用类比和图解来解释复杂概念，侧重于直观理解。避免过度沉溺于深奥的数学证明，而是聚焦于算法的实际应用场景。
- **强结构化**：利用 Markdown 的列表、表格和任务清单来组织内容。每个算法章节必须包含统一的标题结构：背景介绍、图形解析、Go 实现、复杂度分析、性能测试。

## 代码实现准则 (Coding Standards)
- **Idiomatic Go**：代码必须符合 Go 语言的惯用法。包括：
  - 使用 `camelCase` 命名变量，`PascalCase` 导出函数/类型。
  - 显式错误处理（虽然算法中通常较少，但对无效输入应返回 `error`）。
  - 简洁的包结构，每个主要分类（如 `sort`, `tree`）作为一个独立的 package。
- **命名一致性**：在算法实现中，遵循通用的变量命名惯例：
  - 索引使用 `i`, `j`, `k`。
  - 边界使用 `low`, `high`, `left`, `right`。
  - 长度使用 `n` 或 `len`。
- **错误处理准则**：明确区分“正常算法流程”与“边界错误检查”。在函数入口处进行必要的 nil 检查或范围检查。

## 图形解析准则 (Visual Identity)
- **Mermaid.js 优先**：所有流程图、树形结构、链表关系图必须使用 Mermaid.js 语法直接嵌入 Markdown 中，以确保可维护性和直接查看的便利性。

## 性能评估准则 (Performance Evaluation)
- **标准化表格**：在每个算法的 README 中，必须提供一个性能测试表格。
- **实测指标**：包含 `ns/op` (执行时间), `B/op` (内存分配), `allocs/op` (分配次数)。
- **文字分析**：不仅展示数据，还需提供简短的文字分析，解释算法在不同数据规模下的表现差异，以及理论复杂度与实测结果的关联。
